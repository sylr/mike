mike-table-volume(1)
====================

NAME
----
mike-table-volume - mike's volume table

SYNOPSIS
--------

   Column   |           Type           |                               Modifiers                                |                    Description                     
 -----------+--------------------------+------------------------------------------------------------------------+----------------------------------------------------
  id_volume | smallint                 | not null default (nextval('volume_id_volume_seq'::regclass))::smallint | volume unique identifier
  state     | integer                  | not null default 1                                                     | state of the volumes, references mike.volume_state
  path      | text                     | not null                                                               | path of the volumes
  used_size | bigint                   | not null default 0                                                     | used size used on the volumes
  max_size  | bigint                   | not null default 0                                                     | max size available on the volumes
  datec     | timestamp with time zone | not null default now()                                                 | creation date off the volumes
  datem     | timestamp with time zone |                                                                        | last modification date of the volumes
  token     | text                     |                                                                        | security token for volume removal

 Indexes:
    "volume_pkey" PRIMARY KEY, btree (id_volume)

 Check constraints:
    "volume_path_check" CHECK (substr(path, 1, 1) = '/'::text AND "substring"(path, '.$'::text) = '/'::text)

 Foreign-key constraints:
    "volume_state_fkey" FOREIGN KEY (state) REFERENCES volume_state(state)

 Referenced by:
    TABLE "xfile" CONSTRAINT "xfile_id_volume_fkey" FOREIGN KEY (id_volume) REFERENCES volume(id_volume) ON DELETE CASCADE

 Options:
    fillfactor=95

DESCRIPTION
-----------
Volumes represents phisical storage area used to store files managed by mike.

Because dropping a volume will drop all its associated xfiles The token column
is used as security for volume removal.
