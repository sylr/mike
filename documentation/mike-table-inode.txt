mike-table-inode(3)
===================

NAME
----
mike-table-inode - mike's inode table

SYNOPSIS
--------

      Column      |           Type           |                        Modifiers                         |                      Description                       
 -----------------+--------------------------+----------------------------------------------------------+--------------------------------------------------------
  id_inode        | bigint                   | not null default nextval('inode_id_inode_seq'::regclass) | inode unique identifier
  id_inode_parent | bigint                   |                                                          | identifier of parent inode
  id_user         | integer                  | not null                                                 | owner of the inode
  state           | smallint                 | not null default (0)::smallint                           | state of the inode, references mike.inode_state
  id_mimetype     | smallint                 | not null                                                 | mimetype of the inode
  name            | text                     | not null                                                 | name of the inode, limited to 255 characters
  path            | text                     | not null                                                 | path of the inode
  treepath        | ltree                    | not null                                                 | treepath of the inode
  ctime           | timestamp with time zone | not null default now()                                   | creation timestamp with timezone of the inode
  mtime           | timestamp with time zone |                                                          | last modification timestamp with timezone of the inode
  size            | bigint                   | not null default 0                                       | size of the inode
  versioning_size | bigint                   | not null default 0                                       | versioning size of the inode

 Indexes:
    "inode_pkey" PRIMARY KEY, btree (id_inode)
    "inode_ctime_btree_idx" btree (ctime) WITH (fillfactor=95)
    "inode_mtime_btree_idx" btree (mtime) WITH (fillfactor=95)
    "inode_id_inode_btree_idx" btree (id_inode) WITH (fillfactor=95)
    "inode_id_inode_parent_btree_idx" btree (id_inode_parent) WITH (fillfactor=95)
    "inode_id_mimetype_btree_idx" btree (id_mimetype) WITH (fillfactor=95)
    "inode_id_user_btree_idx" btree (id_user) WITH (fillfactor=95)
    "inode_name_btree_idx" btree (name) WITH (fillfactor=95)
    "inode_path_btree_idx" btree (path) WITH (fillfactor=95)
    "inode_treepath_gist_idx" gist (treepath) WITH (fillfactor=95)

 Check constraints:
    "inode_name_check" CHECK (name <> ''::text AND length(name) <= 255)
    "inode_path_check" CHECK (substr(path, 1, 1) = '/'::text)
    "inode_treepath_check" CHECK (nlevel(treepath) <= 24)

 Foreign-key constraints:
    "inode_id_inode_parent_fkey" FOREIGN KEY (id_inode_parent) REFERENCES inode(id_inode) ON DELETE CASCADE
    "inode_id_mimetype_fkey" FOREIGN KEY (id_mimetype) REFERENCES mimetype(id_mimetype)
    "inode_id_user_fkey" FOREIGN KEY (id_user) REFERENCES "user"(id_user)
    "inode_state_fkey" FOREIGN KEY (state) REFERENCES inode_state(state)

 Referenced by:
    TABLE "inode" CONSTRAINT "inode_id_inode_parent_fkey" FOREIGN KEY (id_inode_parent) REFERENCES inode(id_inode) ON DELETE CASCADE

 Options:
    fillfactor=90

DESCRIPTION
-----------
Extends each vfs item of mike: file and directory.
